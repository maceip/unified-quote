Run cards for software that asks for power.

Runcard turns this repo's Stage 0 and Stage 1 attestation chain into a small object an app can check before it releases a token, secret, deploy right, filesystem tool, or customer data.

The technical core is already here: a TEE build receipt, a runtime receipt, attested TLS, hardware quote verification, and a chain walk back to the source identity called Value X.

implementation

The card is backed by the existing proof chain.

Runcard is not a new claim on top of the codebase. It is the developer surface for the chain implemented under v2/.

Stage 0 / build

Runs in TEE hardware, freezes the source tree, runs the build, hashes the artifact, computes Value X, and emits EAT CBOR.

  • source hash before build
  • read-only source snapshot
  • artifact hash after build
  • TEE quote over EAT binding bytes
Stage 1 / runtime

Loads the Stage 0 receipt, verifies it, recomputes Value X from disk, generates a TLS key inside the TEE, and collects a fresh runtime quote.

  • previous attestation hash
  • runtime quote bound to TLS key
  • attested TLS certificate
  • chain continuity check
Client / gate

Fetches the running service, extracts the EAT from the TLS cert, verifies the platform quote, checks the SPKI binding, walks the previous receipt, then applies project policy.

  • Intel TDX, AMD SEV-SNP, AWS Nitro paths
  • pinned vendor roots
  • stable Value X across stages
  • allow or deny privilege
where it matters

Use it at the boundary where software gets authority.

Most developers do not wake up caring about attestation. They do care when a model-controlled workflow is about to get a write token, a deploy job, a shell, or production data. Runcard makes that boundary checkable. 

const verdict = await runcard.verify(target, {
  source: "github.com/acme/support-flow",
  ref: "refs/heads/main",
  requireReviewedCommit: true
});

if (verdict.ok) {
  await tools.enable("write:issues");
}
tools:
  verify_runtime
  explain_receipt
  should_release_secret
  verify_artifact

deny:
  - shell.exec
  - database.write
  - browser.autofill
- uses: maceip/unified-quote/v2/action@main
  with:
    source: .
    cmd: npm ci && npm test && npm run build

runcard gate --receipt proof-receipt.json --policy .runcard.yml
